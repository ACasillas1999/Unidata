Documentación Técnica - Proyecto Unidata

1. Descripción General

Unidata es un portal centralizado de gestión y sincronización de catálogos (especialmente artículos) entre múltiples sucursales distribuidas.

El sistema actúa como un "hub" que permite consolidar la información de una Base de Datos Maestra (db_master) y replicar/validar el estado de los artículos en diversas bases de datos locales (sucursales).

2. Arquitectura del Sistema

El proyecto está construido sobre Laravel 11 y utiliza una arquitectura de bases de datos distribuidas.

  • Portal Central: Orquestador principal que gestiona las conexiones y la interfaz de usuario.
  • Base de Datos Maestra (Master): Contiene la verdad única sobre los productos.
  • Bases de Datos de Sucursales: Instancias externas a las que el sistema se conecta dinámicamente.

3. Modelo de Datos (Entidades Clave)

Branch (Sucursales)

Almacena las credenciales de conexión para cada sucursal.

Campos: code, name, db_host, db_user, db_password, db_database, status.

DbMasterArticle (Artículos Maestros)

Fuente de verdad del catálogo universal.

Campos: clave, descripcion, unidad_medida, precio_lista, costo_venta.

MatrizHomologacion (Matriz de Estado)

Consolida en una sola tabla el estado de presencia de un artículo en todas las sucursales conectadas.

4. Módulos Principales

  • Gestión de Artículos: Carga masiva de CSV con pre-validación y logs de auditoría.
  • Motor de Sincronización: Procesamiento en segundo plano optimizado para entornos XAMPP/Windows.
  • Global Downloads (GDC): Generación asíncrona de reportes pesados.

5. Stack Tecnológico

  • Backend: PHP 8.2+ / Laravel 11.
  • Frontend: Interfaz moderna con CSS Variables nativo.
  • Infraestructura: Multi-conexión dinámica a MariaDB/MySQL.
